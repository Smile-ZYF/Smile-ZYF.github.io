title: 使用hexo搭建个人博客步骤
date: 2019-10-22 10:11:04

本文章教给大家如何使用hexo搭建个人博客这个是我反复尝试、查阅资料后总结出的，为了能让你们更加快速的看到，我放弃了中午的午休给你们写了这篇博客，并且在室友电脑上也成功配置。想想都欠我些什么哈哈！首先此教程基本抛弃了老师发给的教程步骤，但是Git和Node.js还是要安装的。

目录第一步：删除因为是重新创建，所以要把之前的blog目录删掉。

第二步：安装框架在hexo目录中打开Git Bash输入代码：npm install hexo-cli -g

该过程根据电脑内存的不同可能会持续30s-60s，耐心等待。

第三步：创建blog输入代码：hexo init blog

该过程也是会持续一段时间。

第四步：切换目录输入代码：cd blog

查看是否切换

第五步：安装组件输入代码：npm install

第六步：生成静态页面输入代码：hexo g

第七步：启动本地服务器输入代码：hexo s 这只是本地博客，下面配置远端博客在命令界面按Ctrl+C结束本地服务器。

第八步：布置deployer目录输入代码：npm install –save hexo-deployer-git

该过程需要一段时间的加载，耐心等待！

第九步：修改_config.yml文件打开hexo根目录下的_config.yml文件，修改其deploy标签的内容(应该在最下面)

repo后面的Frightings修改成你们自己的名字！冒号后面跟个空格！修改后输入代码：hexo clean清理缓存，
然后输入代码：hexo g重新布置静态页面。

第十步：布置远端博客输入代码：hexo d

然后他会让你告诉他，你是谁~（Please tell me who you are）
接着咱们就告诉他，咱是谁~输入代码：git config –global user.email “youremail@qq.com“
注意！ 这边youremail@qq.com不是让你照搬抄上！是让你写上你自己的邮箱！接着输入代码：git config –global user.name “yourname”
同样，这边yourname不是让你照搬抄上！是让你写上你自己的用户名。然后依次输入代码:hexo clean清理下缓存，hexo g生成静态页面，hexo d布置远端。

第十一步：查看效果然后在网站中输入你的博客地址yourname.github.io就会出现你的博客。

第十二步：写博客输入代码：hexo new “我的第一篇博客”

这上面写出了你的博客路径，找到他进行编辑。

打开后：

上面有个tags标签,目前没什么用，先删除，要不然可能会编译错误。
编辑完之后：

第十三步：查看效果十二步完成后，依次输入代码：hexo clean清理缓存，输入代码：hexo g生成静态页面，可以发现里面的INFO项中加载了自己写的博客。

输入代码：hexo d 缓存到GitHub。然后再刷新一下自己的博客地址，可以发现自己写的博客已经显示了。

如果需要继续添加文章，也是相同的步骤和代码。

Java面向对象知识点总结

第二章 类和对象的创建

1.面向对象的概念
面向对象的思想是把一个项目，一件事情分成更小的项目，或者说分成一个个更小的部分，
每一部分负责什么方面的功能，最后再由这些部分组合而成为一个整体。

2.类的创建

3.创建对象
类名 对象名 =new 类名()；
对象名.方法名()；对象名.属性
4.构造方法：方法名和类名相同，没有返回值

5.构造方法的重载：在同一个类中，方法名相同，参数个数，顺序，类型不相同的构造方法

6.构造方法的执行：
new 类名（）；—>执行构造方法
super（）；—>执行了父类的构造方法
7.static静态修饰符
静态修饰符可以修饰属性，方法，代码块
类名.属性 类名.方法名
8.Java内存（栈和堆，static，方法区，gc（垃圾回收机制））

第三章 面向对象三大特征—封装

1.封装的概念
封装是指将对象的状态信息隐藏在对象内部，不允许外部程序直接访问对象内部信息
而是通过该类所提供的方法来实现对内部信息的操作访问
2.封装的步骤
（1）把属性私有化（private）
（2）对这些属性设置get和set方法
（3）调用处理这些方法
3.封装的作用：避免一些用户不合理的操作
4.修饰符
（1）public—>公共的：在一个项目中都可以访问，权限最高
（2）private—>私有的：只能在本类中进行访问，权限最低
（3）protected—>安全的，受保护的：同一个包中才能进行访问，继承中子类可以访问父类
（4）default—>默认的：同一个包中能访问

第四章 面向对象的三大特征—继承

1.继承的概念
子类继承父类的属性，方法。
2.继承的关键字：extends
3.继承的语法：
public class 类名 extends 父类{}
4.使用

5.父类构造方法调用：super（）；
调用父类的构造方法必须在子类中调用
必须在子类构造方法的第一行
6.重写@Override
必须继承
方法名一致，参数个数，顺序，类型全部一致
